# Knex — Desafio Técnico 2026.2

Este repositório contém os cases técnicos para as áreas **Backend**, **Frontend** e **Fullstack**. Escolha o case de acordo com a vaga e envie o link do(s) repositório(s) com sua solução.

---

## Por área

### Backend

Resolva o case em [`knex-backend-case/`](./knex-backend-case/).

Sistema simples de controle de estoque e simulação de vendas de produtos, com autenticação, CRUDs e simulação de funcionamento de um gateway de pagamentos para realizar a venda dos produtos.

---

### Frontend

Resolva o case em [`knex-frontend-case/`](./knex-frontend-case/).

Site de exibição de informações de países, utilizando a integração com a API RestContries, para exibir as informações dos paises em forma de lista e também detalhadamente.

---

### Fullstack

Resolva **os dois cases** acima: o case de backend com o sistema de produtos e vendas e o frontend com o sistema de listagem de paises e consumo de API externa.

---

## Entrega

- **Não há formato obrigatório.** Você pode criar um repositório novo, fazer fork deste repositório (forma mais recomendada) ou usar qualquer outra forma de hospedar seu código.
- A entrega é feita **enviando o link dos repositórios** de acordo com o case:
  - **Backend:** link do repositório com a API
  - **Frontend:** link do repositório com o frontend
  - **Fullstack:** link(s) do(s) repositório(s) — pode ser um único repo com backend + frontend ou repositórios separados
